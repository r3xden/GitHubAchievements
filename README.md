## What this is

GitHub Achievements are the small badges that appear on a profile's **Achievements** tab — quiet proof of things you did: a pull request merged, a discussion answered, a repository that people starred. Some are still earnable today. A few are frozen in time, awarded once for programs that no longer run. This repository keeps a clean record of both kinds, in the order GitHub introduced them.

Two documents do the actual work:

| Document | Purpose |
|---|---|
| **[`ACHIEVEMENTS.md`](./ACHIEVEMENTS.md)** | The full catalogue — every badge, oldest to newest, with its earnable status and tier structure. Read this first if you want the map. |
| **[`OBTAIN.md`](./OBTAIN.md)** | The walkthrough — concrete, step-by-step reasoning for how to earn each obtainable badge without gaming the platform. Read this if you want the route. |

## A note on intent

Achievements are a side effect of doing real, useful things on GitHub — reviewing code, fixing bugs, answering someone's question, sharing work worth starring. This repo documents them for people who are curious how the system works, not as an invitation to farm hollow pull requests against your own repositories. The `OBTAIN.md` guide is written with that distinction in mind.
